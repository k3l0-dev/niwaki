# Vocabulary book

The curated design surface, generated from `domain/vocabulary.yaml` and the schema-derived reference map — the same tables the runtime uses, so this book cannot drift from the code.

Every design is rooted at `polUni` (`design()`); the pages below cover one domain each.  Anything outside this vocabulary stays reachable through `.mo(Class, ...)` and `bind_dn(alias=dn)`.

**Root makers** (available on `design()` and, via implicit pop, from any cursor):

- `.tenant(name, **attrs)` → `fvTenant`
- `.infra(**attrs)` → `infraInfra`
- `.fabric(**attrs)` → `fabricInst`
- `.controller(**attrs)` → `ctrlrInst`
- `.phys_dom(name, **attrs)` → `physDomP`
- `.l3_dom(name, **attrs)` → `l3extDomP`

**Domains under `uni`**

- `phys_dom(name).bind(vlan_pool=...)` → `infraRsVlanNs` (DN flavor, target `fvnsVlanInstP`)
- `l3_dom(name).bind(vlan_pool=...)` → `infraRsVlanNs` (DN flavor, target `fvnsVlanInstP`)
